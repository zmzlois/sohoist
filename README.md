# Sohoist

Sohoist is a private introduction network for low-stakes dating, matched by real-life vibe.

Members build a private intro brief, invite trusted referrers, and receive thoughtful introductions through people who know them. The product is intentionally not a swipe app: no public marketplace, no infinite browsing, no algorithmic stranger feed.

## What this repo contains

This is a pnpm + Turborepo monorepo for the Sohoist web app, native app, shared Convex backend, and brand assets.

```txt
apps/
  web/                 Next.js 16 App Router app
  native/              Expo SDK 55 app with Expo Router

packages/
  backend/             Convex schema, queries, mutations, and actions
  assets/              Shared brand images, screen mocks, and fonts

scripts/
  sync-env.sh          Copies common local env values into app env files before dev
```

## Product surface

- Membership application and approval flow
- Voice onboarding that turns answers into a private intro brief
- Pencil-sketch portrait generation for privacy-first profile sharing
- Reward pool setup as a seriousness signal
- Trusted referrer invites and referral submission
- Candidate intro pages and shared profile previews
- Admin review tools for membership applications

## Stack

- Turborepo and pnpm workspaces
- Next.js 16, React 19, Tailwind CSS v4
- Expo SDK 55 and Expo Router
- Convex for backend functions, database, and file storage
- NextAuth credentials auth for the web app
- Lightweight native auth context backed by AsyncStorage
- OpenAI for voice transcription/profile extraction
- Gemini image generation for private pencil sketches

No Clerk setup is required. Some legacy field names may still exist in the schema for compatibility, but current local auth is handled by NextAuth credentials on web and the native auth context in `apps/native/src/auth.tsx`.

## Project structure

### `apps/web`

The web application is a Next.js App Router app.

Important areas:

- `src/app/page.tsx` - public marketing/home page
- `src/app/sign-in/page.tsx` - credentials sign-in page
- `src/app/dashboard/*` - member workspace: brief, photo, reward, privacy, referrals, introductions
- `src/app/apply/page.tsx` and `src/app/application-status/page.tsx` - application flow
- `src/app/admin/page.tsx` - admin application review
- `src/app/intro/[token]/page.tsx` and `src/app/invite/[token]/page.tsx` - shared invite/intro surfaces
- `src/auth.ts` - NextAuth credentials configuration
- `src/proxy.ts` - route protection via the NextAuth proxy export
- `src/app/globals.css` - Tailwind v4 theme and Sohoist design tokens

### `apps/native`

The native application is an Expo Router app.

Important areas:

- `src/app/(auth)` - signed-out routes
- `src/app/(app)` - signed-in member routes
- `src/app/intro/[token].tsx` and `src/app/invite/[token].tsx` - public token-based flows
- `src/screens/*` - screen implementations
- `src/auth.tsx` - native auth context
- `src/theme.ts` - native design tokens
- `src/assets/` - bundled fonts and native assets

### `packages/backend`

The backend package contains the Convex app.

Important areas:

- `convex/schema.ts` - database schema
- `convex/users.ts` and `convex/session.ts` - user lookup and role helpers
- `convex/applications.ts` and `convex/admin.ts` - membership application flow
- `convex/profile.ts` and `convex/voice.ts` - intro brief and voice extraction
- `convex/photos.ts` - photo upload, storage, and sketch generation
- `convex/referrers.ts`, `convex/referrals.ts`, and `convex/introductions.ts` - referral loop
- `convex/rewardPools.ts` - reward pool state
- `convex/sharing.ts` - privacy-safe shared profile reads

Convex generated files live in `packages/backend/convex/_generated/`. They are generated by Convex tooling and should not be hand-edited.

## Quick start

Install dependencies:

```sh
pnpm install
```

Start or connect the Convex backend:

```sh
pnpm --filter @packages/backend setup
```

Create app env files from the examples:

```sh
cp apps/web/.example.env apps/web/.env.local
cp apps/native/.example.env apps/native/.env.local
```

Set the Convex URLs in both app env files:

```sh
# apps/web/.env.local
NEXT_PUBLIC_CONVEX_URL=

# apps/native/.env.local
EXPO_PUBLIC_CONVEX_URL=
```

Then run everything through Turbo:

```sh
pnpm dev
```

This starts the Convex backend, the Next.js web app, and the Expo app through their package scripts.

## Environment variables

### Web app

`apps/web/.env.local`

```sh
NEXT_PUBLIC_CONVEX_URL=
AUTH_SECRET=
SOHOIST_AUTH_PASSWORD=
SOHOIST_AUTH_EMAILS=
ADMIN_EMAIL=
```

Notes:

- `AUTH_SECRET` is required by NextAuth.
- `SOHOIST_AUTH_PASSWORD` is the shared credentials password for non-admin local sign-in.
- `SOHOIST_AUTH_EMAILS` is an optional comma-separated allowlist.
- `ADMIN_EMAIL` signs in as admin without the shared password.

### Native app

`apps/native/.env.local`

```sh
EXPO_PUBLIC_CONVEX_URL=
EXPO_PUBLIC_AUTH_BASE_URL=
EXPO_PUBLIC_SOHOIST_AUTH_PASSWORD=
EXPO_PUBLIC_ADMIN_EMAIL=
```

Notes:

- `EXPO_PUBLIC_AUTH_BASE_URL` should point to the web app when the native app should also try the web NextAuth credentials flow.
- The native app also supports local password verification with `EXPO_PUBLIC_SOHOIST_AUTH_PASSWORD`.

### Convex backend

Set these in Convex when using the relevant AI features:

```sh
OPENAI_API_KEY=
GEMINI_API_KEY=
GEMINI_IMAGE_MODEL=
```

`OPENAI_API_KEY` powers voice transcription and structured brief extraction. `GEMINI_API_KEY` powers pencil-sketch portrait generation. `GEMINI_IMAGE_MODEL` is optional and defaults in code.

```sh
pnpm --filter native-app ios     # run the native app on iOS

```

## Deployment

The web app is configured for Vercel. `apps/web/vercel.json` runs Convex deploy and then builds the Next.js app with the deployed Convex URL.

For a manual deploy flow:

```sh
cd packages/backend
pnpm exec convex deploy --cmd 'cd ../../apps/web && pnpm build' --cmd-url-env-var-name NEXT_PUBLIC_CONVEX_URL
```

For iOS distribution:

```sh
pnpm --filter native-app testflight
```

## Development notes

- Install dependencies in the package that uses them:

  ```sh
  pnpm --filter web-app add package-name
  pnpm --filter native-app add package-name
  pnpm --filter @packages/backend add package-name
  ```

- Keep web and native design tokens aligned between `apps/web/src/app/globals.css` and `apps/native/src/theme.ts`.
- Keep product copy discreet, warm, and privacy-first. Avoid dating-app or marketplace language.
- Do not commit local secrets. App env files are local-only.
- If Convex API types are missing or stale, run `pnpm --filter @packages/backend dev` to regenerate them.
